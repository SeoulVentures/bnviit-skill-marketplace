# 주간 PMO 보고 작성 (weekly-pmo-report)

비앤빛 안과 프로젝트의 한 주간 진행 데이터를 받아 경영/오너 보고용 **주간 PMO 보고 초안**을 작성한다.
이슈를 등급으로 분류하고, 수치는 **입력 근거만** 사용하며(추정 금지), KPI 목표값은 공개 문서에 박지 않고 **knowledge 참조**한다.

## 언제 사용하나

- 한 주간 트랙별 진행 상황·작업지시(task) 상태를 모아 정기 PMO 보고를 만들 때.
- 진척 지연·리스크·의사결정 요청 사항을 등급별로 정리해 오너/센터장에게 올릴 초안이 필요할 때.

## 입력 → 출력

- **입력**: 진행 데이터.
  - 트랙(워크스트림) 목록과 각 트랙의 상태(진행/지연/완료/보류 등).
  - 작업지시(task) 상태(담당 역할·기한·완료 여부·블로커).
  - (선택) 직전 주 보고, KPI 실측치(입력으로 제공된 값만).
- **출력**: 주간 PMO 보고 초안(아래 출력 형식). 각 항목에 **입력 근거(출처) 인용**, 근거 없으면 **abstain**.

## 이슈 등급 분류 (🔴/🟠/🟡/🟢)

입력 데이터의 근거에 따라서만 분류한다. 기준이 모호하면 보수적으로 한 단계 높게 표기하고 사유를 적는다.

| 등급 | 의미 | 분류 기준(예시) |
|---|---|---|
| 🔴 위험(Critical) | 일정/품질/안전에 즉시 영향, 오너 의사결정 필요 | 핵심 마일스톤 미달·기한 초과·안전/의료/법무 리스크·해소 불가 블로커 |
| 🟠 경고(High) | 단기 개입 없으면 위험으로 전이 | 기한 임박+미완료·반복 지연·자원/승인 대기 장기화 |
| 🟡 주의(Watch) | 모니터링 필요, 자체 관리 가능 | 경미한 지연·의존성 대기·경계선 진척 |
| 🟢 정상(On-track) | 계획대로 진행 | 기한 내 진행/완료, 블로커 없음 |

## 절차 (SOP)

1. **입력 파싱**: 트랙·작업지시 상태를 표준 형태로 정리한다(트랙명·상태·기한·담당 역할·블로커).
2. **근거 확인 / abstain**: 보고 항목의 수치·상태는 **입력에 명시된 근거가 있을 때만** 기재한다.
   근거가 없거나 모호하면 **임의 추정하지 말고 abstain**(예: "입력 데이터에 근거 없음 — 확인 필요"로 표기)한다.
3. **등급 분류**: 위 표 기준으로 각 트랙/이슈를 🔴/🟠/🟡/🟢로 분류하고, 분류 사유를 근거와 함께 적는다.
4. **KPI 대비(knowledge 참조)**: KPI 실측치는 입력값만 사용한다. **KPI 목표값은 본 공개 문서에 적지 않고**
   `knowledge`(로컬·gitignored)의 KPI 정의를 참조한다. 본문에는 지표 슬러그와 `<knowledge 참조>` placeholder만 둔다.
5. **다음 주 계획·리스크·의사결정 요청 정리**: 입력 근거 범위 내에서 정리한다(없는 계획을 지어내지 않는다).
6. **초안 반환(HITL)**: 결과는 **초안**이며, 외부 공유·발송·확정은 사람 승인(오너/PMO 검수, ag-08-pmo-reviewer 게이트)을 거친다.

## 가드레일

- **추정 금지 / 입력 근거만**: 모든 수치·상태·진척률은 입력에 명시된 값만 사용한다. 외삽·추정·임의 보간 금지.
  근거가 없으면 **abstain**(공란 또는 "근거 없음 — 확인 필요").
- **출처 인용**: 각 항목은 어떤 트랙/작업지시 입력에서 도출되었는지 출처를 함께 표기한다.
- **KPI 목표값 비포함**: KPI 목표값(숫자+단위)을 공개 문서에 직접 기술하지 않는다 → `<knowledge 참조>` placeholder.
- **실명·기밀 금지**: 담당자·에스컬레이션 대상은 **역할명만**(예: 센터장, PMO, 트랙 리드). 개인 실명·이메일·연락처는
  적지 않고 필요 시 knowledge 참조. 환자 데이터·개인 식별정보는 PMO 보고에 포함하지 않는다.
- **HITL(사람 우선)**: 본 산출물은 **초안**이다. 보고 확정·외부 공유·관련 비가역 조치(발송·예약변경·금전)는
  사람 승인 없이 자동 실행하지 않는다.
- **재현 가능성**: 동일 입력이면 동일 분류가 나오도록, 등급 기준과 근거를 명시적으로 적는다.

## 출력 형식

```
# 주간 PMO 보고 (YYYY-MM-DD ~ YYYY-MM-DD)

## 1. 한 주 요약
- 전체 진행 상태(등급 분포): 🔴 n · 🟠 n · 🟡 n · 🟢 n
- 핵심 하이라이트 / 주요 변화 (입력 근거 인용)

## 2. 트랙별 진행 현황
| 트랙 | 등급 | 상태/진척 (입력 근거) | 블로커 | 다음 액션 |
|---|---|---|---|---|
| <트랙명> | 🟢 | <입력에 근거한 상태> (출처) | - | <계획> |

## 3. 이슈 / 리스크 (등급순)
- 🔴 <이슈> — 사유·영향·필요 의사결정 (출처)
- 🟠 <이슈> — ...

## 4. KPI 현황
| 지표(슬러그) | 실측치(입력값) | 목표값 | 대비 |
|---|---|---|---|
| <kpi-slug> | <입력값 또는 근거 없음> | <knowledge 참조> | <knowledge 참조 기준> |

## 5. 다음 주 계획 (입력 근거 범위)
- ...

## 6. 의사결정 요청 (오너/센터장 — 역할명만)
- ...
```

## 예시

**입력(요약):**
- 트랙 A(상담 자동화): 작업지시 3건 중 2건 완료, 1건 기한 초과(블로커: 외부 승인 대기).
- 트랙 B(케어 스케줄): 전 작업지시 기한 내 진행.
- KPI: 주간 응대율 실측치는 입력에 제공됨, 목표값은 미제공.

**출력 초안(발췌):**
> ## 2. 트랙별 진행 현황
> | 트랙 | 등급 | 상태/진척 (입력 근거) | 블로커 | 다음 액션 |
> |---|---|---|---|---|
> | 트랙 A 상담 자동화 | 🟠 | 작업지시 3건 중 2건 완료, 1건 기한 초과 (입력: track A) | 외부 승인 대기 | 승인 에스컬레이션(역할: PMO) |
> | 트랙 B 케어 스케줄 | 🟢 | 전 작업지시 기한 내 진행 (입력: track B) | - | 계획대로 진행 |
>
> ## 4. KPI 현황
> | 지표(슬러그) | 실측치(입력값) | 목표값 | 대비 |
> |---|---|---|---|
> | weekly-response-rate | (입력 제공값) | `<knowledge 참조>` | `<knowledge 참조>` |
>
> ## 6. 의사결정 요청
> - 🟠 트랙 A: 외부 승인 지연 해소 위한 센터장 개입 요청 (출처: track A 블로커)

> 참고: KPI 목표값은 공개 문서에 포함하지 않으며, 확정 보고는 PMO 검수(ag-08-pmo-reviewer)와 사람 승인을 거칩니다.
